WaveTapC v1.5.0.26. Консольный вариант GUI-утилиты WaveTap_Qt.

Формат: wavetapc [ОПЦИИ] [<file> или <database>] [<to_file> или <to_folder>]

Если WaveTapC запущена без опций, то программа покажет пользовательский
интерфейс по умолчанию.
WaveTapC пытается прочитать опции, не заданные в командной строке, из файла
isiTap.ini. В опциях, в которых задаётся имя файла, для разделения можно
использовать как "\", так и "/". Опции, заданные в командной строке, имеют
более высокий приоритет, чем опции в ini-файле. Опции -?, -s, <file>,
<database>, <to_file> и <to_folder> могут быть только в командной строке.

ОПЦИИ (позиция не имеет значения, исключая опции -? и -lang, а также в случае,
когда файл используется как источник данных):
  -?        - Подсказка, должна быть первой в командной строке, показывает
              подсказку об опции, следующей за ней, например: -? -s или -? -?
              Если за -? нет никакой опции, то показывается общая справка
              (этот текст).
              Чтобы получить справку на русском, опцию -lang нужно поставить
              перед -?. Например, чтобы получить русскую справку о самой -lang,
              опции командной строки должны выглядеть так: -lang=ru -? -lang
              Если русские символы отображаются в консоли неверно, можно
              сменить кодировку по умолчанию (см. справку по -lang);
  -lang     - Язык интерфейса и справки. Чтобы получить справку на русском,
              -lang должна быть первой в командной строке, во всех остальных
              случаях её позиция не имеет значения. По умолчанию: англ.;
  -s        - Описание запроса данных. Сервер данных, список запрошенных
              станций и каналов, а также запрошенный период времени.
              Сервер должен использовать один из протоколов:
              ISI, XFER, SEEDLINK, ARCLINK;
  -d -dest  - Получатель данных. Поддерживаются 3 получателя: БД WSG,
  -write_to   CSS3 (WFDISC + файл(ы) с данными), файл(ы) Mini-Seed.
              О допустимых значениях этой опции см. "-? -d".
              По умолчанию: wfdisc;
  -bo -outputbyteorder
            - Порядок байт результата. В случае -d=wsg игнорируется.
              По умолчанию: be (Big Endian);
  -ugi -usegi -useglobalini
            - В случае -d=wsg: использовать глобальный WSG.INI.
              По умолчанию: да;
  -cssstuc  - Для CSS3: В файле WFDISC записывать код станции большими буквами.
              По умолчанию: нет;
  -csschuc  - Для CSS3: В файле WFDISC записывать код канала большими буквами.
              По умолчанию: нет;
 -lddatefmt - Для CSS3: Формат LDDATE в файле WFDISC. По умолчанию: 0;
  -wfdfmt   - Для CSS3: тип чисел в файлах с данными
              (i4, i2, f8, f4, a0). По умолчанию: i4;
  -msonems  - Для Mini-Seed: данные всех станций/каналов пишутся в один файл.
              По умолчанию: да;
  -msenc    - Для Mini-Seed: алгоритм упаковки (STEIM2, STEIM1, INT32).
              По умолчанию: STEIM2;
  -fixst2   - Для Mini-Seed: пытаться исправить аномальные перепады значений,
              что позволит алгоритму STEIM2 упаковывать без ошибок.
              По умолчанию: нет.

WaveTapC может или читать данные с сервера, или использовать в качестве
источника данных файл WFDisc или файл Mini-Seed. С указанным -d это значит, что
WaveTapC может, например, конвертировать файл Mini-Seed в формат CSS3 и
наоборот или преобразовать файлы CSS3 из одного формата в другой (например,
"i4" в "f8").

<file>, <database>, <to_file> и <to_folder> ДОЛЖНЫ указываться без префиксов,
таких как "-", "--" или "/". Подробнее см. "-? file", "-? db" or "-? database",
"-? to_file" и "-? to_folder".

Чтобы читать данные с сервера, используйте опцию -s. Чтобы читать данные из
файла(ов) CSS3 / Mini-Seed НЕ указывайте -s, а вместо этого укажите имя файла
WFDisc или Mini-Seed в параметре <file> и, по желанию в одном из следующих
параметров укажите <to_file> или <to_folder>. Порядок параметров здесь имеет
значение! <file> должен идти первым, после него <to_file>/<to_folder>. Между
ними могут быть другие опции.

------ Если данные нужно читать с сервера: ------

Пример 1: wavetapc -d=m U:\Data\3\akh
-s=192.168.1.20:18000:sl:!GO_akh:hhz:00:hhe::2019-05-18-22-29-20:24h

Эта команда прочитает данные с сервера SEEDLINK, адрес 192.168.1.20, порт
18000. Будет запрошены следующие данные: код сети "GO", код станции "akh",
код канала "hhz", Lcode "00", а также канал "hhe", пустой Lcode, начиная с
2019-05-18 22:29:20, за 24 часа.
Данные будут помещены в файл формата Mini-Seed с именем U:\Data\3\akh.mseed
Следует заметить, что кол-во и имена файлов Mini-Seed также зависят от опции
-msonems.
Примечание для Linux и bash: символ "!" является специальным для bash, поэтому
его использование вызывает ошибку. Можно вместо "!" использовать "#" или
предварительно отключить такое поведение bash командой "set H+" в консоли.

Пример 2: wavetapc -d=wsg odbc:WSG_aaa:::1:U:\Data\3
-s=192.168.1.20:18000:sl:!GO_akh:hhz:00:hhe::2019-05-18-22-29-20:24h

Читает данные с того же сервера, что в предыдущем примере, но записывает
данные в БД WSG типа ODBC (ODBC-источник "WSG_aaa", пустые пользователь и
пароль (заменятся на "WSG"), use_tend = 1 (true), папка с данными "U:\Data\3").

Пояснение.
Опции -s и -d, и параметр <file>/<database> должны ВСЕ быть указаны в
командной строке.
<file> нужно указывать, если -d - CSS3 или Mini-Seed. Это путь и имя файла
wfdisc или Mini-Seed, заданное так же, как оно выбирается в GUI-диалоге
в WaveTap_Qt при нажатии кнопки "Получить данные".
Расширение (.wfdisc, .mseed) указывать не обязательно. Подходящее расширение
будет добавлено автоматически, если его нет.

<database> нужно указывать, если -d=wsg. Это полное описание подключения к
БД WSG.

Если обе опции (-s & -d) И <file>/<database> (т.е., все 3 из них) указаны в
командной строке, то данные будут немедленно запрошены у сервера -s, помещены в
расположение -d, и программа завершит работу.

------ Если данные нужно читать из файлов CSS3/Mini-Seed: ------

Пример 1: wavetapc -d=m U:\Data\3\akh.wfdisc

Эта команда преобразует файл U:\Data\3\akh.wfdisc и связанные с ним файлы
данных в файл формата Mini-Seed U:\Data\3\akh.mseed. Файл Mini-Seed будет
помещён в ту же папку, что исходный файл - U:\Data\3\
Следует заметить, что кол-во и имена файлов Mini-Seed также зависят от опции
-msonems.

Пример 2: wavetapc -d=w U:\Data\3\akh.mseed U:\Data\444\

Эта команда преобразует файл U:\Data\3\akh.mseed в файлы формата CSS3 с
заголовочным файлом формата WFDisc U:\Data\444\akh.wfdisc. Все файлы CSS3 будут
помещены в папку U:\Data\444\

Пример 3:
wavetapc U:\Data\3\akh.wfdisc -d=w U:\Data\444\akh_f8.wfdisc -wfdfmt=f8

Эта команда преобразует файл U:\Data\3\akh.wfdisc и связанные с ним файлы
данных (формата "i4", int) в CSS3, но с данными в другом формате - "f8",double.
Все преобразованные файлы CSS3 будут помещены в директорию U:\Data\444\,
заголовочный файл WFDisc будет с именем akh_f8.wfdisc

Пояснение.
Опция -s не указывается. Опция -d может быть указана в командной строке или
прочитана из ini-файла. <file> должен быть указан (без префикса!),
<to_file>/<to_folder> - по желанию. Чтобы задать <to_folder>, а не <to_file>,
в конце её должен быть символ "/" или "\". Если такого символа в конце нет, то
считается, что это <to_file>.
ВНИМАНИЕ: при конвертировании CSS3 в Mini-Seed, имена файлов данных должны быть
в определённом формате: "NN_STA.CHAN.LCODE.<остальное>".
NN - код сети (1 - 3 символа), после него символ "_" (подчёркивание), STA - код
станции, точка ("."), код канала, точка, LCODE, точка, остальная часть имени.
Здесь требуются NN, CHAN и LCODE, т.к. они нужны в Mini-Seed, а в файле WFDisc
они чётко не выделяются.

<file> определяет файл(ы) CSS3/Mini-Seed, которые будут использованы в качестве
источника данных. Тип файла определяется по расширению (".wfdisc" - формат CSS3
и ".mseed" или без расширения - формат Mini-Seed).
Опция -d по-прежнему определяет получателя данных, но <database> указать
нельзя. Если -d=wsg, то настройки подключения к БД WSG можно указать только в
ini-файле.
Если -d - не "wsg", то <to_file> определяет файл-приёмник, а <to_folder> -
папку-приёмник результата. Если в <to_file> не указано расширение, то оно
добавится автоматически, в зависимости от -d.
Если <to_file> или <to_folder> не указана, то результат будет записан в папку,
взятую из <file>, с тем же именем файла + ".res" и с подходящим расширением.
Другие опции, такие как -bo и -msenc, также применимы.

Опции командной строки и ini-файла.
-------------------------------------------------------------------------------
###?###
ОПЦИЯ -?

ФОРМАТ:  -? [-_другая_опция_]
ПРИМЕР:  -? -s

Получить справку о "_другой_опции_". -? должен быть первым в командной строке,
а справка выдается об опции, следующей за ним.
Опция -? может быть второй только в том случае, когда нужно показать справку на
русском языке. Тогда первой идёт опция -lang с указанием языка и, если нужно,
кодировка, которая используется в консоли. Кодировку можно задать, если русские
символы отображаются неверно, т.е., кодировка консоли отличается от той, что
программа использует по умолчанию. Подробнее - см. справку по -lang.
Также -lang можно задать в ini-файле настроек, тогда в командной строке её
можно не указывать.
Если _другая_опция_ не задана, то показывается общая справка.

Эта опция НЕ может быть указана в ini-файле.
###?###

-------------------------------------------------------------------------------
###lang###
ОПЦИЯ -lang

ФОРМАТ:  -lang=язык[:кодировка]
ПРИМЕРЫ: -lang=ru:UTF-16
         -lang=ru:1251
         -lang=ru:cp866
         -lang=1

Язык интерфейса и справки. Чтобы получить справку на русском, опция -lang
должна быть первой в командной строке, во всех остальных случаях её позиция
не имеет значения. По умолчанию - en.
Язык может быть "en"/"0" (английский) или "ru"/"1" (русский).
Кодировка для языка - это кодировка для консоли. Она может принимать
одно из следующих значений (фактически, все, которые поддерживает Qt):
1.  IBM 866       2.  KOI8-R        3.  KOI8-U       4.  Windows-1250 до 1258
5.  UTF-8         6.  UTF-16        7.  UTF-16BE     8.  UTF-16LE
9.  UTF-32        10. UTF-32BE      11. UTF-32LE     12. Apple Roman
13. Big5          14. Big5-HKSCS    15. CP949        16. EUC-JP
17. EUC-KR        18. IBM 850       19. IBM 874      20. ISO 2022-JP
21. JIS X 0201    22. JIS X 0208    23. MuleLao-1    24. ISO 8859-1 до 10
25. ROMAN8        26. Shift-JIS     27. TIS-620      28. ISO 8859-13 до 16
29. TSCII         30. WINSAMI2      31. GB18030-0
32. Iscii-Bng, Dev, Gjr, Knd, Mlm, Ori, Pnj, Tlg, and Tml
Для номеров 1, 4, 15, 18, 19 можно указать просто число или "cp" + число.
"cp" (английскими буквами) - code page.
Если кодировка не указана, то используется кодировка консоли по умолчанию:
для Windows: IBM 866 (русская кодировка DOS-а);
для других ОС (MacOS, Unix-ы): UTF-8.
Также, если -lang задана в ini-файле настроек, тогда в командной строке её
можно не указывать.

В ini-файле: "lang" в разделе [-config-]
###lang###

###d###
ОПЦИЯ -d -dest -write_to

ФОРМАТ:  -d=<получатель>
ПРИМЕР:  -d=mseed или -d=m

Получатель данных. Получателем может быть:

1. CSS3 (wfdisc + файл(ы) с данными). В качестве <получатель> допустимы строки
"w", "c", "wf", "css", "css3", "wfd", "wfdisk", "wfdisc"
(любая из этих строк означает, что получателем будет CSS3);

2. Файл(ы) Mini-Seed. В качестве <получатель> допустимы строки
"m", "s", "ms", "mseed", "seed", "miniseed", "mini-seed";

3. База данных WSG. В качестве <получатель> нужно указывать "wsg".

Для немедленного, автоматического получения данных от сервера опция -d должна
быть указана вместе с опцией -s и <file>/<database> (т.е., все три параметра
должны быть правильно заданы в командной строке).

В ini-файле: "write_to" в разделе [-config-]
###d###

-------------------------------------------------------------------------------
###s###
ОПЦИЯ -s

ФОРМАТ:  -s=<адрес>:<порт>:<тип>:[<пользователь>:<пароль>:]
![<сеть-1>_]<станция-1>:<канал-1>:<lcode-1>[...:<канал-N1>:<lcode-N1>]
[...:![<сеть-X>_]<станция-X>:<канал-1>:<lcode-1>[...:<канал-NX>:<lcode-NX>]]
:<начало периода>:<конец периода> | <длина периода>

ПРИМЕРЫ: -s=192.168.1.10:18000:sl:!GO_akh:hhz:00:hhe::2018-05-18-11-20-00:10m
         -s=192.168.1.10:18001:a:<aaa@mail.org>::#PR_ggg:hhz:00:#PR_ddd:hhz::
2018-05-18-11-20-00:2018-05-19-02-00-00

В этой опции задаётся полное описание запроса данных: сервер, список станций и
каналов, период времени - всё в одну строку. Отдельные элементы разделяются
двоеточием.
Сервер указывается в начале строки и может быть только один.
<адрес> - IP-адрес или сетевое имя сервера;
<порт> - порт, на котором сервер отвечает на запросы. Обычно, 39136 для
протокола ISI, 14002 для XFER, 18000 для SEEDLINK и 18001 для ARCLINK.
Возможны другие значения, они зависят от настроек сервера;
<тип> - тип сервера, а именно его протокол. В настоящий момент поддерживаются
4 протокола: ISI, XFER, SEEDLINK и ARCLINK.
<тип> может принимать следующие значения:
для ISI: "i", "isi";     для XFER: "x", "xfer";
для SEEDLINK: "s", "sl", "slink", "seedlink";
для ARCLINK: "a", "al", "alink", "arclink".
<пользователь> и <пароль> указываются ТОЛЬКО для типа ARCLINK.
Это пользователь и пароль, с которыми будут делаться запросы к серверу ARCLINK.
Далее указывается хотя бы один код станции <станция> и коды её каналов <канал>
с соотв. <lcode>.
Перед кодом станции обязательно ставится восклицательный знак "!" или
решётка "#".
Примечание для Linux и bash: символ "!" является специальным для bash, поэтому
его использование вызывает ошибку. Можно вместо него использовать "#" или
предварительно отключить такое поведение bash командой "set H+" в консоли.
Также, в зависимости от типа сервера, между "!"/"#" и <станция>
может быть код сети <сеть>, после которого ставится подчёркивание "_".
Код сети нужен для серверов SEEDLINK и ARCLINK, но не нужен для ISI и XFER.
Также код сети (и, возможно, остальные коды) чувствителен к регистру, и если,
например, не подошёл код "go_", можно попробовать "GO_". Рекомендуется
предварительно свериться со списком доступных на сервере кодов.
Если lcode пустой или состоит из пробелов, <lcode> нужно указывать пустым,
например: ...!GO_akh:hhe::... (между двумя двоеточиями - пустой lcode).
Далее могут быть указаны коды других станций и каналов этого сервера.
Далее обязательно указывается начало периода времени, за который будет сделана
выборка данных. Период общий для всех указанных станций и каналов.
<начало периода> имеет формат: yyyy-mm-dd-hh-nn-ss. Все поля разделены
знаком "-" (минус). Здесь yyyy - год из 4 цифр, mm - месяц из 2 цифр,
dd - день из 2 цифр, hh - час из 2 цифр, nn - минута из 2 цифр,
ss - секунда из 2 цифр.
Далее обязательно указывается либо <конец периода>, либо <длина периода>.
<конец периода> имеет формат, аналогичный, но можно указать только время:
[yyyy-mm-dd-]hh-nn-ss. Если часть "yyyy-mm-dd-" (дата) опущена, она принимается
равной дате начала периода. Пример: 2019-05-12-12-32-45:17-00-00
Если указана <длина периода>, то конец периода считается как
<начало периода> + <длина периода>. Формат длины периода: n<единица>.
Здесь n - целое число, <единица> может принимать следующие значения:
"h" - час, "m" - минута, "s" - секунда. Пример: 2019-05-12-12-32-45:1h

Эта опция НЕ может быть указана в ini-файле.
###s###

-------------------------------------------------------------------------------
###file###
Параметр <file>

ФОРМАТ:  <путь и имя файла>

ПРИМЕРЫ: U:\Common\dev\3\www
         /home/user/data/www
         D:\data\www.wfdisc

Если опция -s указана:

Если в опции -d указан получатель CSS3 или Mini-Seed, то этот параметр
(без "=" и без префиксов типа "--", "-", "/") считается именем файла(ов), в
которые будут записаны данные соотв. формата. Количество файлов зависит от
других опций в ini-файле. Если расширение имени файла (.wfdisc, .mseed) не
указано, то соотв. расширение будет автоматически добавлено. В примерах,
приведённых выше, к "www" будет добавлено ".wfdisc" или ".mseed".
Параметр <file> аналогичен имени файла, которое выбирается через GUI-диалог в
WaveTap_Qt при сохранении данных в CSS3 или Mini-Seed.

Если опция -s НЕ указана:

Этот параметр считается именем файла-источника данных в формате WFDisc или
Mini-Seed. Формат определяется по расширению (".wfdisc" - WFDisc из CSS3,
".mseed" или без расширения - Mini-Seed).
ВНИМАНИЕ: при конвертировании CSS3 в Mini-Seed, имена файлов данных должны быть
в определённом формате: "NN_STA.CHAN.LCODE.<остальное>".
NN - код сети (1 - 3 символа), после него символ "_" (подчёркивание), STA - код
станции, точка ("."), код канала, точка, LCODE, точка, остальная часть имени.
Здесь требуются NN, CHAN и LCODE, т.к. они нужны в Mini-Seed, а в файле WFDisc
они чётко не выделяются.
Параметр <to_file>/<to_folder>, если он указан, должен идти после <file>.

Этот параметр НЕ может быть указан в ini-файле.

###db###
Параметр <database>

ФОРМАТ1: <тип БД>:<адрес>:<порт>:<источник>:<пользователь>:<пароль>:
<использовать tend>:<папка данных>

ФОРМАТ2: <тип БД>:<источник>:<пользователь>:<пароль>:
<использовать tend>:<папка данных>

ПРИМЕР:  mysql:192.192.192.192:3456:WSG_aaa:userrr:passs:1:U:\Common\dev\3

Если в опции -d указан получатель WSG, то этот параметр (без "=" и без
префиксов типа "--", "-", "/") считается описанием БД WSG, в которую будут
записаны данные. В обеих форматах все поля обязательны.
Пустой <адрес> будет заменён на "localhost". Пустые <источник>, <пользователь>
и <пароль> будут заменены на "WSG". Пустая <папка данных> будет заменена на
"D:\DATA" или "~/DATA".

<тип БД> может быть одним из следующих: 1. "odbc"; 2. "sqlite"; 3. "sqlite2";
4. "mysql" или "mariadb"; 5. "postgres" или "postgresql";
6. "oracle" или "ora"; 7. "interbase" или "firebird";
8. "sybase" или "mssql" или "tds"; 9. "ibmdb2"; 10. "mdb".

"sqlite" - это SQLite версии 3 и выше.
"mdb" - это фактически ODBC, но вместо ODBC-источника в качестве <источника>
задаётся файл MS Access с расширением MDB.
Эти типы соответствуют типам подключений, которые поддерживает Qt. Реальная их
поддержка в WaveTapC (и WaveTap_Qt) зависит от того, была ли собрана WaveTapC
с использование Qt, в которой есть поддержка конкретных типов подключения, а
также, если Qt используется в виде разделённых библиотек (.dll, .so),
установлен ли в ОС драйвер-клиент от производителя конкретной СУБД.
Для <тип БД> 1, 2, 3 и 10 (ODBC, SQLite, SQLite2, MS Access MDB) применяется
ФОРМАТ2 (т.к., это файловые СУБД), для остальных - ФОРМАТ1 с указанием адреса
и порта сервера БД.
<источник>. Для ODBC - имя ODBC-источника, для SQLite SQLite2 и MDB - путь и
имя файла БД, для остальных имя базы данных на сервере.
<пользователь> и <пароль> должны быть указаны (пустыми?), даже если не нужны
(в случае MS Access и SQLite).
<использовать tend> - использовать ли поле TEND в операциях с БД WSG.
Может быть "true", "1" или "yes" - тогда TEND будет использоваться;
"false", "0", "no" - не будет.
<папка данных> - папка, куда будут записываться файлы с данными волновых форм
для БД WSG. Может быть локальной или сетевой.

Этот параметр НЕ может быть указан в ini-файле.

###bo###
ОПЦИЯ -bo -outputbyteorder

ФОРМАТ:  -bo=<порядок байт>
ПРИМЕР:  -bo=be

Порядок байт результата, т.е. файлов с данными, как CSS3, так и Mini-Seed.
Эта опция связана с опцией -wfdfmt. В случае -d=wsg игнорируется.
<порядок байт> может принимать значения:

1. "be", "big_endian", "big-endian", "bigendian" - порядок байт таких
архитектур, как SPARC. Старший байт числа идёт в памяти первым;

2. "le", "little_endian", "little-endian", "littleendian" - порядок байт таких
архитектур, как Intel. Младший байт числа идёт в памяти первым.

В ini-файле: "outputbyteorder" в разделе [-config-]

ОПЦИЯ -lddatefmt

Для CSS3: Формат LDDATE в файле WFDISC. <формат> принимает значения:

ОПЦИЯ -wfdfmt

         -wfdfmt=float

Для CSS3: <формат> - тип чисел в файлах с данными, может принимать значения:

Для Mini-Seed: алгоритм упаковки, может принимать значения:
