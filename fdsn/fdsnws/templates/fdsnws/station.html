What is the nodata=404 URL parameter?

Usage

Network Network code or virtual network code, wildcards or lists.
Station Up to 5 character station code, wildcards or lists.
Location 2 character location identifier, wildcards or lists.
Channel 3 character channel codes, wildcards or lists.
Wildcards The network, station, location and channel fields may contain the * or ? wildcards with the following meaning:
  • * – matches zero to many characters
  • ? – matches a single character
Network wild card is not supported when level=resp
Lists The network, station, location and channel fields may contain a comma-separated list, examples include BHE,BHZ or BH*,LH*.
Start Date Station information must start on or before the date specified.
Specify dates as: YYYY-MM-DD (e.g. "2010-06-14")
End Date Station information must end on or before the date specified.
Specify dates as: YYYY-MM-DD (e.g. "2010-06-14")
Level Choose the level of granularity for the information returned, can be network, station, channel or response level.
Format Specify output format.
Start Before Station information must have started before the date specified.
Start After Station information must have started after the date specified.
End Before Station information must have ended before the date specified.
End After Station information must have ended after the date specified.
Updated After Return events updated since the given date.
Sensor Select matching sensor descriptions. Sub-string match is case-insensitive and may be specified with wildcards and lists. The match is against the string in the Sensor::Description schema location.
Lat/Lon Box Return events within a lat/lon range.
Lat/Lon Radius Return events a given distance (in degrees) of a location.
Include Restricted Channels Specify whether results should include information relating to restricted channels.
Include Data Availability Specify if results should include information about time series data availability at the channel level.
Include Comments Specify if results should include station and channel comments. By default this is true.
Line up availabilities with available data Retrieve only metadata that is likely to match available time series data.
Nodata If no data is returned by the service, return a 404 response rather than the default 204.
Learn more about HTTP response codes.
